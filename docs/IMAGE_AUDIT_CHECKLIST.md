## 1. 검수 등급

| 등급 | 의미 | 운영 |
|------|------|------|
| **PASS** | 기술적으로 정확, 즉시 사용 | `status: reviewed` — 노출 허용 |
| **MINOR_FIX** | 라벨·색·캡션·화살표만 수정 | 수정 후 PASS — 노출 허용 |
| **MAJOR_FIX** | 설치 위치·방향·구성 일부 오류 | 수정 완료 전 **노출 금지** |
| **REGENERATE** | 계측 원리 자체 오류 | 처음부터 재생성 — **노출 금지** |
| **DELETE** | 신뢰도 훼손 | `rejected/` 보관 — **영구 금지** |

**운영 허용:** PASS, MINOR_FIX 만 (`resolveImage`·`audit:images`).

## 2. 검수 기록 형식 (필수)

각 이미지는 [IMAGE_REVIEW_LOG.md](./IMAGE_REVIEW_LOG.md)에 다음 항목을 남긴다.

```text
Image ID:
파일명:
사용 페이지:
관련 계측기:
검수 등급:
기술 오류:
설치 위치 오류:
방향 오류:
용어 오류:
수정 지시:
재생성 프롬프트:
검수자:
검수일:
```

레지스트리 동기화:

```bash
# scripts/image-review-registry.json 해당 id 수정 후
node scripts/generate-image-assets.mjs
npm run audit:images
```
