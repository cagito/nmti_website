## 3. 통합 파이프라인 (4단계)

### Phase A — 색인·갭 분석 (주 1회)

| 단계 | 작업 | 산출 |
|------|------|------|
| A1 | `npm run catalog:book-pdf` | `source-index.md` 갱신 |
| A2 | `docs/` ↔ `image-knowledge/` 주제 매핑 표 | 갭 목록 (PDF만 있는 주제 / docs만 있는 주제) |
| A3 | ATS·와이어·침하 등 **정책 키워드** grep | [127](./127-변위계측-자동광파기-남용방지-및-와이어식-우선-표준.md) 같은 **신규 P1 표준** 후보 |

### Phase B — 상위 표준 승격 (정책 확정 시)

1. **한 줄 정책** + 표 + Cursor 지시문 작성 (`docs/12x-*.md`)
2. **INSTRUMENTATION §3.x** 에 5~15줄 **요약 블록** + 링크
3. **51 DP/DIS** 또는 **63 정확성** 에 검수 항목 1~3개
4. **AGENTS.md** + `.cursor/rules/*.mdc` (에이전트 강제)
5. **image-knowledge/0n-*.md** (PDF 근거 + 실행 규칙)

### Phase C — Figure·프롬프트 하향 반영

| 우선순위 | 대상 | 작업 |
|----------|------|------|
| C1 | **hero** · category 개요 (001·011·015·100) | callout 재배치 · ATS 축소 |
| C2 | **정책 위반 redline** (090·101·015) | redline v3 · 프롬프트 v4 |
| C3 | **PASS지만 관성 ATS** (005·012·013·061) | optional/보조 문구 |
| C4 | **content-data** 본문 | 「와이어식」「자동 센서」 우선 서술 |
| C5 | registry `notes` | `DISP-ATS-01` · 재작도 사유 |

### Phase D — 검증·잠금

```bash
npm run validate:image-knowledge   # image-knowledge 내부 링크·구조
npm run validate:terminology
npm run audit:images
npm run verify:content             # 재작도 중
npm run verify:local               # 배포 전
```
