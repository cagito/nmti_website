### 현재 문제 (기술 검토)

1. **C1** 건물↔벽체 `이격거리 L` = **허공** — 배면 토사 없음

### IMG-046 — IoT 게이트웨이 구성도 (MAJOR_FIX)

```text
IoT 게이트웨이 구성도를 수정한다.

현재 그림은 IoT 게이트웨이가 프로토콜 변환, 데이터 버퍼링, 보안전송, 통신모듈 기능을 모두 수행하는 것으로 보이지만, 데이터로거와 게이트웨이의 역할 경계가 불명확하다.

수정사항:
1. 데이터로거는 센서 신호 수집·저장·1차 변환 장치로 표시한다.
2. IoT 게이트웨이는 통신 중계, 프로토콜 변환, 보안터널, 재전송 관리 장치로 표시한다.
3. 로거 저장 데이터와 게이트웨이 임시 버퍼를 구분한다.
4. 통신 장애 시 로거 로컬 저장과 게이트웨이 재전송 흐름을 표시한다.
5. 시간 동기화는 로거 기준인지 서버 기준인지 명확히 한다.
6. 보안은 암호화뿐 아니라 인증, 장비 식별, 접근권한으로 표시한다.
7. 서버 수신 후 데이터 검증과 경보 판단은 플랫폼 영역으로 분리한다.
8. 게이트웨이가 계측값을 해석하거나 관리기준을 판정하는 것처럼 표현하지 않는다.

금지:
- 게이트웨이를 데이터로거와 같은 장치처럼 표현하지 말 것.
- 게이트웨이가 모든 이상판정을 수행하는 것처럼 표현하지 말 것.
```
