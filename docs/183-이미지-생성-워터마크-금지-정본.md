## 1. 원칙 (필수)

**GenerateImage · 외부 AI · CAD · 인간 일러스트 — 생성·export 원본에 NMTI 로고·워터마크·브랜드 마크를 넣지 않는다.**

| 단계 | 로고·워터마크 |
|------|----------------|
| **생성·source 저장** | **금지** — 깨끗한 도면만 |
| **등록 후·배포 전** | **일괄 후처리** — 아래 §2 |
| **SPA·SEO 노출** | 후처리본 + 클라이언트 보호(§3) |

이중 합성(생성 시 AI 로고 + 스크립트 로고)을 막기 위한 **절대 규칙**이다.

---

## 2. 일괄 후처리 (유일한 합성 경로)

| 명령 | 용도 |
|------|------|
| `watermark-figures.bat` | Windows 일괄 적용 (권장) |
| `npm run watermark:figures` | `scripts/apply-figure-watermark.py` |
| `npm run audit:figure-watermark` | manifest·미적용 검수 |

- **에셋:** `assets/images/brand/nmti-watermark.png`
- **로직:** `scripts/lib/watermark_stamp.py`
- **원본:** `assets/images/technology/source/` — **워터마크 없는 정본** 유지
- **배포:** `assets/images/technology/` — 후처리본(필요 시)

`register:figure` · `sync:images` **는 워터마크를 넣지 않는다.** 배포 직전 또는 일괄 갱신 시 §2 스크립트를 실행한다.

---

## 3. 생성 시 금지 목록

- NMTI 로고·워터마크·유사 로고·흰색 로고 박스
- 우하단 임의 마크·「NMTI」 텍스트 로고
- 제조사·브랜드 로고(별도 METHOD 규칙)
- AI가 그린 **placeholder** 로고(후처리와 중복)

**선택(권장):** 우하단 **여백**만 확보 — **로고 픽셀은 그리지 않음**.

---

## 4. 프롬프트·에이전트

- **§36 §1.0** 10줄 블록 · **§2.1 Prefix** · Negative에 동일 금지
- **ImageWorks** `prompts/IMG-###_*.md` — `<!-- watermark-sync:v1 -->` 블록 (`npm run sync:prompt-watermark`)
- **에이전트:** `.cursor/rules/agent-figure-generation.mdc` · [16](./16-기술자료-이미지-에이전트-SVG-생성-금지.md)

```text
No NMTI logo, no watermark, no brand mark, no company logo in the generated image.
Watermark is applied later by scripts/apply-figure-watermark.py only.
```

## 5. 검수

- **visualReview PASS** 전: 생성본에 로고·워터마크 픽셀 **없음** 확인

- 배포 전: `audit:figure-watermark` (해당 파이프라인 사용 시)
