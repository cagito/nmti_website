## §1 IMG-047 — 태양광 전원 (MAJOR_FIX · SOLAR-SIZE-01)

```text
태양광 전원 시스템 구성도를 수정한다.

현재 그림은 태양광 패널, 충전제어기, 배터리, 데이터로거를 단순 직렬 연결로 표현하고 있다. 실제 현장 계측 전원은 부하 산정, 자율운전일, 배터리 용량, 충방전 보호, 서지보호, 접지, 저전압 차단이 핵심이다.

수정사항:
- 태양광 패널, 충전제어기(MPPT/PWM), 배터리, DC 분전, 부하, 데이터로거, 통신모뎀을 분리 표시한다.
- 부하는 센서 여자전원, 데이터로거, 통신모뎀, 히터/팬 등으로 구분한다.
- 배터리 용량은 Ah, DoD, 자율운전일, 일사량 조건으로 산정한다고 표시한다.
- 저전압 차단(LVD), 과충전·과방전 보호, 퓨즈, 차단기, 서지보호기를 추가한다.
- 패널 방향, 음영, 설치각, 도난·파손 보호를 표시한다.
- 접지는 배터리 하나가 아니라 함체·SPD·패널프레임·로거 접지를 구분한다.

금지:
- 태양광 패널을 연결하면 장기 무정전 운전이 보장되는 것처럼 표현하지 말 것.
- 부하 산정 없이 배터리 12V만 표시하지 말 것.

1920x1080, Korean engineering one-line diagram, white background.
```

## §2 IMG-048 — LTE M2M (MAJOR_FIX · LTE-BUF-01)

```text
LTE M2M 통신 구성도를 수정한다.

현재 그림은 센서, 데이터로거, LTE 모뎀, 서버, 웹·모바일을 단순 직렬 흐름으로 표현하고 있다. 실제 원격계측 통신은 통신두절, 재전송, 보안, APN, 방화벽, 서버 수신확인, 로컬 저장이 중요하다.

수정사항:
- 데이터로거와 LTE 모뎀의 역할을 분리한다.
- 로거에는 로컬 저장 기능을 표시한다.
- 통신두절 시 로컬 저장 후 재전송하는 흐름을 추가한다.
- LTE망, APN, VPN 또는 사설망, 방화벽, 서버 수신 API를 표시한다.
- 서버 ACK 또는 수신확인 절차를 추가한다.
- 웹·모바일은 서버 데이터를 조회하는 클라이언트로 표현한다.
- 통신상태, RSSI, 데이터 지연, 누락 패킷을 모니터링 항목으로 넣는다.

금지:
- LTE 모뎀이 데이터를 해석하거나 경보를 판단하는 것처럼 표현하지 말 것.
- 웹과 모바일이 직접 현장 로거를 제어하는 것처럼 단순화하지 말 것.

1920x1080, block diagram, Korean labels.
```

## §3 IMG-049 — 변위 그래프 (MAJOR_FIX · DISP-GRAPH-01)

```text
변위 그래프 예시를 수정한다.

현재 그림은 옹벽 변위를 단순 양방향 기준선으로 관리하는 것처럼 표현한다. 변위 계측은 기준점 안정성, 측정 방향, 누적변위, 변위속도, 구조물 회전 여부를 함께 검토해야 한다.

수정사항:
- 변위 방향을 “전방 변위”, “후방 변위” 등 현장 기준축으로 명확히 표시한다.
- 기준점 또는 고정측이 안정한지 표시한다.
- 변위계가 측정하는 값은 상대변위임을 표시한다.
- 그래프에는 누적변위와 변위속도 검토 항목을 함께 표시한다.

금지:
- 단일 변위계 값으로 옹벽 전체 안정성을 판단하는 것처럼 표현하지 말 것.
- ±기준선을 모든 현장 일반 기준처럼 표시하지 말 것.
```

## §4 IMG-050 — 침하 그래프 (REGENERATE · GRAPH-PRED-01)

```text
침하 그래프 예시를 전면 수정한다.

현재 그림은 측정 침하 추세를 단순 외삽하여 예측침하로 표시하고 있다. 침하 예측은 성토단계, 압밀도, 간극수압 소산, 지반정수, 쌍곡선법 또는 Asaoka법 등 해석 방법을 구분해야 하며 단순 선형 외삽으로 표현하면 안 된다.

수정사항:
- 제목을 “침하-시간 계측 및 예측 개념도”로 수정한다.
- 측정값과 예측값을 명확히 구분한다.
- 예측값은 단순 추세연장이 아니라 “해석법 적용 결과”로 표시한다.
- 성토단계, 방치기간, 추가성토, 프리로딩 해제 시점을 그래프에 표시한다.
- 잔류침하, 최종침하, 압밀도 개념을 추가한다.
- 기준점은 영향권 밖 안정 기준점으로 별도 표시한다.
- 관리기준은 현장별 기준으로 표시한다.

금지:
- 측정값 몇 개를 단순 연장해 최종침하로 표시하지 말 것.
- 침하판 상단이나 연장봉을 기준점처럼 표현하지 말 것.

1920x1080, settlement-time graph, Korean.
```

## §5 IMG-051 — 간극수압 소산 (MAJOR_FIX · PIEZO-DISS-01)

```text
간극수압 소산 그래프를 수정한다.

현재 그림은 성토 단계마다 간극수압이 일정하게 상승하고 부드럽게 소산되는 이상화된 패턴만 표현한다. 실제 해석에서는 초기 정수압, 과잉간극수압 Δu, 심도별 반응, 배수조건, 성토속도, 계측기 위치가 중요하다.

수정사항:
- Y축을 “과잉간극수압 Δu(kPa)” 또는 “간극수압 u(kPa)” 중 하나로 명확히 한다.
- 초기 정수압과 성토로 인한 과잉간극수압을 구분한다.
- 성토 단계별 하중 증가 시점을 표시한다.
- 소산 곡선은 심도별로 다를 수 있음을 표시한다.
- 배수재 유무, 배수거리, 투수성 조건을 주석으로 넣는다.
- 관리기준은 고정 수평선이 아니라 성토 안정관리 기준과 연동된다고 표시한다.

금지:
- 모든 성토단계에서 동일한 수압 상승·소산 패턴을 일반화하지 말 것.
- 간극수압 소산만으로 압밀 완료를 단정하지 말 것.

1920x1080, pore pressure dissipation chart.
```

## §6 IMG-052 — 하중 변화 (REGENERATE · LOAD-STAGE-01)

```text
하중 변화 그래프를 전면 수정한다.

현재 그림은 버팀보 하중이 굴착단계마다 증가하고, 동일한 정상·주의·경고 기준선을 모든 단계에 적용하는 것처럼 표현한다. 버팀보 하중 관리는 단계별 설계축력, 설치시점, 프리로드, 온도영향, 급증·급감 여부를 함께 봐야 한다.

수정사항:
- 각 버팀보 단별 설계축력 또는 관리기준을 별도로 표시한다.
- 프리로드 또는 초기하중 설정 시점을 표시한다.
- 굴착단계, 지보재 설치, 해체 시점을 구분한다.
- 하중 증가뿐 아니라 급격한 하중 감소도 위험 신호로 표시한다.
- 온도 변화에 따른 하중 변동 가능성을 주석으로 넣는다.
- 단일 기준선 대신 단계별 관리기준 또는 설계축력 대비 비율로 표현한다.
- 그래프에는 누적 하중보다 실측축력, 변화율, 설계대비율을 함께 검토한다고 표시한다.

금지:
- 전체 버팀보 하중에 동일한 수평 기준선을 적용하지 말 것.
- 하중 증가만 위험이고 하중 감소는 안전한 것처럼 표현하지 말 것.

1920x1080, load vs excavation stage chart.
```

## §7 IMG-053 — 진동 그래프 (MAJOR_FIX · VIB-GRAPH-01)

```text
진동 계측 그래프를 수정한다.

현재 그림은 이벤트 번호와 시간축을 혼합하고, PPV를 단일 점 그래프로만 표시한다. 진동 계측은 3축 성분, 최대입자속도, 주파수, 지속시간, 이벤트 시간, 측정 위치가 함께 필요하다.

수정사항:
- X축을 “이벤트 번호” 또는 “시간” 중 하나로 통일한다.
- PPV는 3축 중 최대성분인지, 합성 PPV인지 명확히 표시한다.
- X, Y, Z 성분별 최대값 또는 대표성분을 구분한다.
- 이벤트 발생시간, 진동원, 측정위치를 표시한다.
- 주파수 성분 또는 우세주파수 검토 항목을 추가한다.
- 관리기준은 구조물 종류, 진동원, 주파수 조건에 따라 달라진다고 표시한다.
- 경보 기준값은 예시 또는 현장별 기준으로 표시한다.

금지:
- PPV 하나만으로 모든 진동 영향을 판정하는 것처럼 표현하지 말 것.
- 가속도계와 진동속도계의 단위를 혼동하지 말 것.

1920x1080, vibration monitoring chart, mm/s.
```

## §8 IMG-054 — 경보 프로세스 (REGENERATE · ALARM-FLOW-01)

```text
경보 단계 프로세스를 전면 수정한다.

현재 그림은 정상, 주의, 경고, 위험, 조치 완료가 단순 선형으로 진행되는 것으로 표현한다. 실제 계측 경보는 데이터 품질검증, 센서 이상, 결측, 지속시간, 변화속도, 현장확인, 해제조건이 포함되어야 한다.

수정사항:
- 경보 판단 전에 데이터 품질검증 단계를 추가한다.
- 센서 이상, 통신두절, 결측은 계측값 초과와 별도 상태로 분리한다.
- 기준 초과는 누적값, 변화속도, 지속시간 조건으로 구분한다.
- 경보 발생 후 현장확인, 계측기 점검, 관련 센서 비교 단계를 추가한다.
- 조치 완료 후에는 경보 해제 조건과 모니터링 강화 단계를 넣는다.
- 위험 단계는 자동으로 조치 완료로 가지 않도록 한다.
- 알림 대상과 승인권한을 구분한다.

금지:
- 측정값이 기준선을 넘으면 곧바로 구조물 위험으로 단정하지 말 것.
- 조치 완료를 경보 프로세스의 자동 종료처럼 표현하지 말 것.

1920x1080, flowchart, Korean, no SF neon UI.
```

## §9 IMG-055 — 모바일 경보 (MAJOR_FIX · MOB-ALARM-01)

```text
모바일 경보 알림 화면을 수정한다.

현재 화면은 기준 초과값과 색상 경보를 보여주지만, 현장 대응에 필요한 데이터 품질, 지속시간, 추세, 관련 센서, 조치이력이 부족하다.

수정사항:
- 알림 카드에 센서 상태, 통신 상태, 배터리 상태를 표시한다.
- 초과값뿐 아니라 초과 지속시간과 최근 변화율을 표시한다.
- 관련 센서 동시 변화 여부를 표시한다.
- 현장명, 공구명, 위치, 센서번호, 계측항목, 단위를 명확히 표시한다.
- “확인”과 “조치” 버튼은 확인자, 시간, 조치내용이 로그에 남는 것으로 표현한다.
- 오경보, 센서점검, 현장확인, 작업중지 등 조치 유형을 구분한다.
- 경보 해제 조건을 표시한다.

금지:
- 색상 알림만으로 현장 조치가 가능하다는 식으로 표현하지 말 것.
- 기준값 초과만으로 위험을 단정하지 말 것.

1080x1920 or 16:9 mobile mockup, realistic monitoring app style.
```

## §10 IMG-056 — 웹 대시보드 (REGENERATE · DASH-STATE-01)

```text
웹 대시보드 구성도를 전면 수정한다.

현재 대시보드는 지도, 센서목록, 그래프, 이벤트로그가 형식적으로 배치되어 있으나 계측 운영에 필요한 좌표, 센서 상태, 단위, 기준선, 시간축, 이벤트 연계가 부족하다.

수정사항:
- 지도에는 실제 현장도 또는 평면도 기반의 센서 위치를 표시한다.
- 센서 목록에는 센서명, 계측항목, 단위, 상태, 최근 수신시간을 표시한다.
- 그래프에는 선택된 센서명, 계측항목, 단위, 시간범위, 초기치, 관리기준을 표시한다.
- 이벤트 로그는 센서 이벤트, 통신 이벤트, 사용자 조치 이벤트를 구분한다.
- 지도 상태색과 센서목록 상태색, 이벤트 로그가 일관되게 연결되도록 한다.
- 결측, 통신두절, 배터리 저전압, 센서 이상 상태를 별도 표시한다.
- 다중 센서 비교 그래프와 관련 센서 동시 변화 확인 기능을 넣는다.

금지:
- 모든 센서를 단순 초록 점으로 표시하지 말 것.
- 단위와 시간축 없는 그래프를 쓰지 말 것.
- 이벤트 로그와 실제 센서 상태가 불일치하는 화면을 만들지 말 것.

1920x1080, web dashboard wireframe, Korean labels.
```
