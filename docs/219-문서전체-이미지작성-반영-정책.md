# 문서 전체 코퍼스 기반 이미지 작성 정책

**목적:** 기술자료 Figure, hero, 설치 개념도, 분석도, 프롬프트, redline, Pillow/CAD 렌더링 작업은 저장소의 PDF·MD 등 모든 문서파일을 검토한 뒤 작성한다.

---

## 1. 적용 범위

다음 작업은 모두 본 정책을 선행한다.

- `assets/images/technology/` 신규 이미지 제작·교체
- `ImageWorks/.../prompts/IMG-###_*.md` 작성·수정
- `scripts/render-*.py` · `scripts/lib/*draw*.py` · SVG/Pillow/CAD 계열 렌더링 코드 수정
- `scripts/image-review-registry.json` · `docs/IMAGE_REVIEW_LOG.md` 판정 변경
- 기술자료 HTML/SEO hero 이미지 연결 변경

---

## 2. 읽어야 하는 문서파일

이미지 작업자는 아래 문서군을 모두 검색·대조한다.

| 구분 | 포함 파일 |
|---|---|
| 루트 정책 | `AGENTS.md`, `README.md`, 루트 `*.md` |
| 기술 문서 | `docs/**/*.md` |
| PDF/HWP 원문 | `book/**/*.{pdf,hwp,hwpx,md,txt,json}` |
| 이미지 지식 | `docs/image-knowledge/**/*.md` |
| ImageWorks | `ImageWorks/**/*.md` |
| 검수·레지스트리 | `docs/IMAGE_REVIEW_LOG.md`, `scripts/image-review-registry.json`, `scripts/figure-production-policy.json` |
| 콘텐츠 소스 | `scripts/content-data/**/*.mjs`, `book/kds-kcs-citation-registry.json` |

`node_modules`, `.git`, `assets/images/**`, `dist/**`, `logs/**`, `backup/**`는 문서 근거에서 제외한다.

---

## 3. 필수 작업 순서

```bash
# 1) 전체 문서 코퍼스 목록 갱신
node scripts/catalog-image-document-corpus.mjs --write

# 2) 기존 계측 이미지 검증
npm run audit:images:strict
node scripts/validate-image-master.mjs

# 3) 이미지 생성·수정 후 재검증
npm run build:images
npm run audit:images:strict
node scripts/validate-image-master.mjs
```

이미지를 그리기 전에는 `docs/document-corpus-manifest.json`에서 관련 문서가 누락되지 않았는지 확인한다.

---

## 4. 이미지별 반영 원칙

각 IMG-ID 작업 전 다음 순서로 문서를 대조한다.

1. `scripts/image-review-registry.json`의 `title`, `caption`, `prohibitedErrors`, `reviewDoc`
2. 해당 `reviewDoc` 및 관련 `docs/###-IMG-###-*.md`
3. `docs/image-knowledge/NN-*.md`
4. `book/` 원문 PDF/HWP에서 유래한 기준 문서 및 인용 레지스트리
5. ImageWorks prompt와 redline 문서
6. 기술자료 본문(`scripts/content-data/**/*.mjs`)의 계측 목적·설치 위치·측정 물리량

---

## 5. 충돌 처리

문서 간 내용이 충돌하면 아래 우선순위를 따른다.

1. `AGENTS.md` 및 본 정책
2. `CANONICAL_DOC_INDEX.md` Tier A/B 문서
3. KDS/KCS 용어·인용 레지스트리
4. `TECHNICAL_IMAGE_STANDARD.md`, `INSTRUMENTATION_DRAWING_RULES.md`
5. `docs/image-knowledge/` 정본
6. IMG별 최신 재생성 기록·review log
7. 과거 계획 문서

충돌이 해소되지 않으면 이미지를 생성하지 말고 `requiresReaudit: true` 또는 별도 수정계획 문서로 보류한다.

---

## 6. 금지

- 특정 MD 몇 개만 읽고 Figure를 생성하는 행위 금지
- PDF 원문 또는 PDF에서 파생된 image-knowledge를 확인하지 않고 단면·센서 위치를 확정하는 행위 금지
- `prohibitedErrors`를 삭제해 PASS처럼 보이게 하는 행위 금지
- 문서 근거 없이 임의 관리기준, 임의 깊이, 임의 센서 배치를 삽입하는 행위 금지
- 내부 검수 코드, PASS/FAIL, P0/P1 같은 문구를 이미지 픽셀에 노출하는 행위 금지

---

## 7. 완료 조건

이미지 작업 완료 보고에는 다음을 포함한다.

```text
- 확인한 문서군: docs / book / image-knowledge / ImageWorks / registry / content-data
- 반영한 핵심 근거: 계측 목적, 설치 위치, 측정 물리량, 기준점, 금지표현
- 실행 검증: catalog-image-document-corpus, build:images, audit:images:strict, validate-image-master
```
