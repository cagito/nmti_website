### 2.1 공학 표준 Prefix (연속 생성 시 문두 고정)

모든 ID별 프롬프트 **맨 앞**에 아래 한 줄을 붙여넣는다.

```text
[Engineering Blueprint Constraints: 2D Cross-section Schematic only. High-contrast line art. Strict Civil Engineering logic. No human brain graphics. No abstract futuristic background. Clear symbol icons with leader lines.]

[P0 MANDATORY — apply before all other rules]
- Building entrance door threshold ON ground surface (GL), never inside soil/fill layers.
- Sensor-type multi-stage inclinometer CASING and groundwater standpipe MUST be **drilled from GL** with **surface borehole opening visible** (protective cap / well cap at ground surface). NOT mid-stratum only, NOT hidden below ground, NOT disconnected from GL.
- Data logger MUST be drawn as a visible field enclosure: light-gray rectangular industrial logger (LCD + front terminal ports) OR logger inside weatherproof box with sensor cables converging — NOT legend text only, NOT empty box, NOT junction-box icon only, NOT PC/server rack.
- Label 「지표면」 explicitly on cross-sections.
- No NMTI logo, no watermark, no brand mark in the generated image — watermark is applied later by watermark-figures.bat only.
```

### 2.4 토질 레이어 해칭 규칙 (Soil Layering Rule — 지중 단면 공통)

지반을 AI에 맡기면 **갈색 덩어리**로 뭉뚱그리는 경우가 많다. **말뚝·가시설·터널·연약지반·주변지반** 등 **지중 단면** 프롬프트 **맨 아래**에 아래 블록을 **그대로** 덧붙인다.

```text
[Soil Layering Rule for AI Cursor]
Represent the underground soil mass as clearly defined horizontal engineering strata:
1. Top layer: Loose sandy silt (dot pattern).
2. Middle layer: Soft marine clay (horizontal dashed lines).
3. Bottom layer: Competent weathered or soft bedrock (cross-hatch diagonal lines).
Ensure the sensors change their installation detail (e.g., bentonite seal vs sand filter) depending on these soil layers.
Label 「지표면」 at ground surface (GL).
```

### 5.2 흔한 공학적 오류 — 즉각 수정 (Troubleshooting)

배치 생성 중 **부분 오류**가 반복되면 §5.1 전에 아래 **EN 수정 명령**을 그대로 붙여넣는다.

#### 오류 1 — 센서가 공중에 부유 (floating icon)

**원인:** AI가 센서를 레이아웃 **아이콘**으로만 인식.

```text
The strain gauge (or tilt meter / load cell) must be physically welded or bolted onto the structure surface. Redraw so the sensor body is tightly attached to the beam flange or wall face, with clear mounting brackets on both ends. No floating icons above the structure.
```

#### 오류 2 — 흙막이 단면에서 토사·굴착 경계 불명

**원인:** retained ground vs excavation void 혼동.

```text
Clearly divide the canvas. The left 50% must be solid layered soil matrix (retained ground). The right 50% must be completely empty excavated space (sole void). The vertical retaining wall must perfectly separate these two zones. Re-apply docs/36 §4.1 left/right rule.
```

#### 오류 3 — 장비·시스템 Figure의 SF·사이버풍

**원인:** 「system」「communication」「datalogger」 → 네온·홀로그램·뇌.

```text
Remove all glowing neon lines, futuristic particle waves, and human brain graphics completely. Replace with standard electrical black line-art inside a grey rectangular control box enclosure. Keep it looking like a real construction site product manual — docs/36 §1.2 negative constraints.
```

### 6.1 대량 생성 QA 체크리스트 (Cursor 배치 렌더링)

수십 개 Figure를 연속 생성할 때 AI가 일반 일러스트로 **회귀**하는 경향이 있습니다. **출력 1장마다** 아래 4가지를 검수합니다.

#### 1. 지반·구조 배치가 엔지니어링 룰에 맞는가?

| 분야 | 좌→우(또는 상→하) 관례 |
|------|------------------------|
| **가시설** | 좌: 원지반+건물 → 우: **굴착 공동** |
| **터널** | 지하 원형/아치 → 지상 **침하 트러프** |
| **연약지반** | 하부 연약층 → 상부 **사다리꼴 성토** |
| **댐** | 좌: **상류 만수위** → 우: **하류** |
| **항만** | 좌: **배면 매립** → 우: **바닷물·조위** |
| **철도** | 노반·궤도 **축단면** — 2레일+침목+道床 (열차·역 금지) |
| **건축/구조** | RC **보·슬래브·벽** detail — 조감도 금지 |

#### 2. 센서 물리적 부착 형상이 타당한가?

| 센서 | 올바른 표현 |
|------|-------------|
| **변형률계** | H형강 flange에 **용접** 원통형 **변형률계** |
| **하중계** | 강선/앵커 **관통 도넛형** center-hole |
| **구조물경사계** | 외벽 **L브래킷** 표면 부착 |
| **다점지중변위계** | 보링 내 **길이 다른 강봉+앵커** (≠ inclinometer) |
| **간극수압계** | **밀폐 filter tip** (≠ 개방 관측공) |

#### 3. 네거티브 제약 유지?

- 사람·스마트폰 · **뇌(Brain)** · SF 빛무리 · cyberpunk → **즉시 §5 또는 §5.1**

#### 4. 선(Line) 중심 도면인가?

- 사실적 흙·바위 **텍스처·그라데이션·그림자** 최소화 → **회색/흰색 라인 아트** · CAD blueprint

## 9. Cursor 이미지 양산 마무리 (Wrap-up)

`dictionary.js` **전 분야·전 nodeId** Figure를 Cursor로 순차 생성할 때, 세션 종료 전 아래 **3항**을 반드시 확인한다. (§6.1 QA · §5.1 Feedback · §5.2 Troubleshooting과 병행)

### 9.1 지시선·기호 중심 (Manual / Specification Style)

- 배경 **그라데이션·질감·사실적 렌더** 금지
- **Symbol + Leader line**으로 센서명·위치·데이터 흐름을 명시 — 시방서·매뉴얼 Figure 톤
- 강조색은 **단색 muted** (water=blue, sensor/vector=red) — 네온·SF UI 금지

### 9.2 단면도 톤앤매너 통일 (2D Cross-section Consistency)

- `fields/*` 지중·굴착·터널·사면·연약지반 → **2D Cross-section** 통일 (§2.1 · §2.4)
- `instruments/*` · `INSTRUMENT_SUBGROUPS` → **Block Diagram / Topology** (§4.8 · §4.13)
- 스크롤 시 **동일 CAD blueprint** 인상 유지 — 3D·풍경화 1장이라도 §5 즉시 재생성

### 9.3 네거티브 필터·3단계 보정

디자인 그래픽·풍경화·뇌·홀로그램이 **조금이라도** 보이면:

1. **§5 Hard Constraint** 전체 재생성  
2. 부분 오류 → **§5.1 Feedback Loop** (구조·센서·선 단순화 3단계)  
3. 반복 실패 → **§5.2 Troubleshooting** + **§2.5 Master Package** 재주입
