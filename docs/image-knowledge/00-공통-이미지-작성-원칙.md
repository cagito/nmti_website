# 공통 이미지 작성 원칙

> **입력 근거:** `book/` 디렉터리 PDF만 사용한다. 본 문서와 주제별 기준서는 PDF 요약이 아니라 Cursor Agent가 따라야 할 **실행 규칙**이다.

## 1. 기본 원칙

- 하나의 Figure는 하나의 기술 목적만 설명한다.
- PDF에 근거 없는 계측기, 센서, 장비, 공법, 브랜드명을 추가하지 않는다.
- 실제 설치 가능 구간과 설치 제한 구간을 구분한다.
- 계측 방향, 변위 방향, 하중 방향, 수위선, 기준선을 혼동하지 않는다.
- 현장 사례를 일반 원칙처럼 과도하게 일반화하지 않는다.
- 불확실한 내용은 이미지에 그리지 않거나 `현장 조건에 따라 상이`로 처리한다.
- 문서 간 충돌 시: 시방서·계측계획서 > 보고서 > 교육·제안 자료. 도면·텍스트 불일치는 `불확실` 표기.

## 2. 스타일 원칙

- 흰색 또는 밝은 회색 배경
- Deep Navy: 구조물, 라이닝, 흙막이, 지반선, 장비 외형
- Teal: 계측점, 측선, 변위 방향, 수위선 등 핵심 기술 요소
- Cool Gray: 보조선, 기준선, 초기 형상, 설명선
- 빨강은 경보도나 위험도 설명이 아닌 경우 사용하지 않는다.
- 네온, 사이버펑크, 캐릭터, 사람, AI 회로, 뇌, 과도한 3D 홍보 이미지 금지

## 3. 출력 원칙

- 운영 이미지는 WebP
- 파일명은 SEO용 풀네임 사용
- 축약 파일명 사용 금지
- PNG 운영본 생성 금지
- SVG 파일 생성 금지
- SVG 문자열, SVG→PNG/WebP 변환 금지

## 4. Cursor Agent 작업 원칙

이미지를 그리기 전에 반드시 다음을 먼저 읽는다.

1. 본 공통 원칙 (`00-공통-이미지-작성-원칙.md`)
2. 해당 주제 이미지 작성 기준 (`01-*.md` …)
3. 해당 이미지 전용 문서 (ImageWorks prompt / redline — 있을 때)
4. 이미지 registry 또는 `js/technology/images.js`

그리기 전 반드시 다음을 출력한다.

- Figure 목적
- 표현 대상
- 설치 가능 구간
- 설치 제한 구간 / 미표현 구간
- 반드시 그릴 요소
- 절대 금지 요소
- 최종 WebP 경로
- 사용할 렌더링 방식 (외부 AI/CAD + 인간 검수 — FT-A/B)

위 요약이 기준서와 충돌하면 이미지를 생성하지 말고 멈춘다.

## 5. PDF 분석 범위

- 분석·인용 대상: `book/**/*.pdf` (보조: 동일 폴더 PNG·추출 txt)
- `book/` 외 문서·웹·학습 데이터를 Figure 근거로 쓰지 않는다.
- 라벨 한글 용어는 `book/KDS-KCS_용어기준.md`와 충돌 시 용어기준을 따른다 (설치 위치·표현 제약은 PDF 우선).

## 6. 현장 사례 vs 일반 원칙

| 구분 | Figure에 반영 방법 |
|------|-------------------|
| KDS/KCS·설치 매뉴얼 반복 내용 | 일반 기준으로 작성 |
| 특정 현장 준공도면·계획서만의 배치 | `현장 사례` — 일반 도면처럼 그리지 않음 |
| 문서 간 차이 | `현장 조건 의존` — 둘 중 하나만 단정하지 않음 |

## 7. 변위·광학계측 (DISP-ATS-01 · ATS-SUB-01)

> **정본:** [127 DISP-ATS-01](../127-변위계측-자동광파기-남용방지-및-와이어식-우선-표준.md) · [206 ATS-SUB-01](../206-자동광파기-지하공사-전용-표현-통합-적용-계획.md) · [09 변위·광학 주제 기준](./09-변위·광학계측-표현-기준.md)

- 변위·처짐·신축이음 Figure의 **기본 수단** = **와이어식 변위계** · LVDT · 침하계 · 내공변위계 등 **센서**.
- **자동광파기(ATS)** = **지반 하부 공사(터널·깊은 굴착)·IMG-042** 맥락만 — **지상 분야 Figure·본문 금지** (206).
- ATS를 그릴 때: 부동점·시준선·프리즘=측점 분리 — 측점에 TS 본체 **금지**.
- 준공·유지관리 맥락 Figure: ATS **생략 또는 (보조)** — 센서형 **주계측**.

## 8. zip 112 WebP 재검수 · PASS 5게이트 (2026-06-29)

> **정본:** [180 재검수 계획](../180-technology-이미지-전수-재검수-수정계획.md) · [181 금지조건](../181-이미지별-계측오류-금지조건-정본.md) · [182 PASS](../182-분야별-계측이미지-PASS-기준.md)

```text
계측은 장비 나열이 아니다.
계측은 특정 공학적 질문에 답하기 위해
측정 항목, 계측기 종류, 설치 위치, 측정 방향, 해석 방법을 정하는 작업이다.
```

PASS 전 **5게이트** (1개라도 NO → PASS 불가): 설치 위치 · 측정 물리량 · 방향 · 기준·축 · 확정/추정/예시.

**대외 금지:** AI 로고 · 워터마크 · PASS/REGENERATE/P0/BRI-01 등 내부 코드 · 범례↔본도 불일치.

**LOGO-01 (생성):** Figure **그릴 때** NMTI·MTi·워터마크·© **금지** — 우하단 **여백만**. 운영 합성은 [183](../183-이미지-생성-워터마크-금지-정본.md) · `watermark-figures.bat` · `npm run watermark:figures` **후처리만**.
