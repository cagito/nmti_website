| 항목 | 이유 |
|------|------|
| `assets/logo.svg` · favicon · nav 로고 | UI 크롬 |
| 썸네일·아이콘·`< 400px` 표시 폭 | 가독성·워터마크 과다 |

### 1.4 크기 판정 규칙 (구현용)

```text
protected =
  (naturalWidth >= 600 OR renderedWidth >= 480)
  AND (matches .img-protected OR .tech-figure OR .tech-seo-hero OR [data-img-protect])
```

서버 합성 대상은 **Figure 마스터 PNG 전종** — 표시 크기와 무관하게 **파일 단위** 워터마크.

## 2. 목표 아키텍처 (2계층)

```text
[1차 · 강함] PNG/WebP 생성 시 우하단 로고 합성 (Pillow / draw.py 공통)
       ↓
[2차 · 억제] HTML/CSS/JS — draggable·contextmenu·선택 차단 + CSS 워터마크 백업
       ↓
[표시] SPA · SEO · lightbox — 동일 정책 클래스
```

| 계층 | 역할 | 우회 난이도 |
|------|------|-------------|
| **1차 합성** | URL로 받은 파일 자체에 로고 | 중 |
| **2차 클라이언트** | 우클릭·드래그·길게누르기 저장 억제 | 낮~중 |
| CSS 오버레이 | 합성 누락분 보완·라이브 재렌더 대비 | 낮 (DevTools로 제거 가능) |

**권장:** 1차+2차 **병행**. 2차만으로는 라이트박스·직링크에 원본 노출.

## 3. 워터마크 디자인

### 3.1 에셋

| 파일 (신규) | 용도 |
|-------------|------|
| `assets/images/brand/nmti-watermark.png` | 우하단 합성용 (**N 원형 + MTi** 또는 가로 축소, **투명 PNG**) |
| `assets/images/brand/nmti-watermark.svg` | 벡터 원본 (Pillow rasterize 입력) |

**제작:** `assets/logo.svg`에서 **N 원형 마크** + `MTi` 축소 또는 회사명 `신계측기술정보` 1줄. 가로 로고 전체(300×52)는 우하단에 과대.

### 3.2 배치 규칙 (WATERMARK-01)

| 항목 | 값 |
|------|-----|
| 위치 | **우하단** — 오른쪽·아래 **margin = max(24px, 2% × min(W,H))** |
| 크기 | 로고 폭 = **min(W×0.12, 180px)** (비율 유지) |
| 불투명도 | **72~85%** (도면 라벨과 겹침 최소) |
| 배경 | 선택 — 반투명 흰/네이비 pill (`rgba(255,255,255,0.55)`) 가독성용 |
| 금지 | 중앙 대형 워터마크 · Figure 기술 라벨 가림 · FT-A 치수선 위 덮음 |

### 3.3 파이프라인 통합

| 경로 | 조치 |
|------|------|
| `scripts/lib/*_draw.py` | 마지막에 `stamp_watermark(im)` 호출 |

## 4. 클라이언트 보호 (우클릭·드래그·저장 억제)

### 4.1 모듈

**신규:** `js/image-protection.js`

```javascript
export function initImageProtection(root = document) { ... }
```

**호출:**

| 페이지 | 시점 |
|--------|------|
| `js/technology/app.js` | `renderContent` 후 · `initFigureLightbox`와 함께 |
| SEO `generate-technology-seo-pages.mjs` | `<script type="module">` import |
| `index.html` (P1) | 대형 스크린샷만 `[data-img-protect]` |

### 4.2 HTML 속성

```html
<img ... draggable="false" class="img-protected" data-img-protect>
```

`content-loader.js` `renderFigure()` · SEO `heroFigureHtml()` 에 **자동 부여**.

### 4.3 이벤트·CSS

| 조치 | 구현 |
|------|------|
| 우클릭 방지 | `contextmenu` → `preventDefault()` (`.img-protected` 및 부모 `figure`) |
| 드래그 방지 | `dragstart` → `preventDefault()` · `draggable="false"` |
| 선택 방지 | `user-select: none` · `-webkit-user-drag: none` on `.img-protected` |
| iOS 길게누르기 | `-webkit-touch-callout: none` (완전 차단 아님) |
| 복사 | `copy` on figure — 선택적 `preventDefault()` |
| 키보드 저장 | **차단 불가** — 범위 밖 |

### 4.4 라이트박스

`figure-lightbox.js` 확대 이미지에도:

- `draggable="false"` · `class="img-protected"`
- dialog 열릴 때 `initImageProtection(dialog)` 재적용
- 합성 워터마크가 1차에 있으면 확대 시에도 **픽셀에 포함**

### 4.5 CSS 워터마크 백업 (2차 표시)

```css
.img-protected-wrap::after {
  content: '';
  position: absolute;
  right: 12px; bottom: 12px;
  width: 72px; height: 24px;
  background: url('/homepage/assets/images/brand/nmti-watermark.png') no-repeat right bottom / contain;
  pointer-events: none;
  opacity: 0.85;
}
```

`renderFigure`에서 `.tech-figure__media-wrap` 래퍼 추가. **1차 합성과 중복 시** CSS 오버레이는 `data-watermark-baked="1"` 일 때 **숨김**.

### 4.6 접근성

- `alt`·`figcaption` **유지** (스크린리더)
- 우클릭 막아도 **키보드·터치 확대(라이트박스)** 는 유지
- 전 페이지 `contextmenu` 금지 **하지 않음** — 보호 대상만
