# 계측 목적 카드 UI·콘텐츠 개편 계획

**수립:** 2026-06-22  
**우선순위:** **P2** (가독성·전문성 · 3종 템플릿 완성도)  
**선행:** [44-3종 템플릿](./44-건설계측-메뉴·템플릿-통합-구현계획.md) Phase E · `content-loader.js` · `css/technology.css`  
**참고 벤치마크:** `sensors/inclinometer/index.html` (`info-card` · `matrix-card` · `definition-box` · 표 혼합)

> **한 줄:** 동일한 `title + body` 박스만 반복되는 **계측 목적(`purpose`)** 섹션을, **3종 템플릿별 카드 변형·시각 계층·센서 칩·콘텐츠 밀도**로 개편한다.

## 1. 현황·문제 진단

### 1.1 렌더링 (단조로움의 원인)

| 항목 | 현재 |
|------|------|
| 렌더러 | `js/technology/content-loader.js` → `purpose` 배열만 `card-grid` + `tech-card` |
| 마크업 | `<h3 class="tech-card__title">` + `<p>{body}</p>` **고정** |
| 스타일 | `css/technology.css` — 16px 패딩·1px 테두리·배경만 (`minmax(220px,1fr)` 그리드) |
| 변형 | **없음** — 분야·센서·시스템 동일 UI |

```137:153:js/technology/content-loader.js
  if (key === 'purpose' && Array.isArray(value)) {
    return (
      '<div class="card-grid">' +
      value
        .map(function (card) {
          return (
            '<div class="tech-card"><h3 class="tech-card__title">' +
            escapeHtml(card.title) +
            '</h3><p>' +
            card.body +
            '</p></div>'
          );
        })
        .join('') +
      '</div>'
    );
  }
```

### 1.2 콘텐츠 (정보 밀도 불균형)

| 구분 | `purpose` 건수(대략) | 특징 |
|------|---------------------|------|
| 분야 카테고리 | 9 × 4~6카드 | 문장형 — 나쁘지 않으나 **시각 구분 없음** |
| 분야 리프 | ~70 | 흙막이 5카드 등 **양호** · 일부 리프는 2~3줄 반복 |
| 센서 | 20 × 3카드 | **과소** — 예: 침하계 「누적·속도를 추적합니다」 수준 |
| 계측 시스템 | 6허브 × purpose | 「구성 요소」 나열 — 카드보다 **칩·아이콘**이 적합 |

**체감:** 지중경사계 **전용 페이지**(`info-card` + 표 + callout) 대비 기술자료 SPA **계측 목적**이 동일 회색 카드 3~5개로만 보여 **단조·저밀도**로 느껴짐.

### 1.3 템플릿별 섹션 라벨 (혼동 요인)

| 템플릿 | kicker | h2 | 동일 `purpose` 키 |
|--------|--------|-----|-------------------|
| 분야 | Measurements | **주요 계측 항목** | ✅ |
| 센서 | Purpose | **측정 목적** | ✅ |
| 시스템 | Components | **구성 요소** | ✅ |

UI는 같고 **의미만 다름** → 시스템 페이지는 카드 그리드가 어색할 수 있음.

## 2. 목표·설계 원칙

### 2.1 목표

1. **한눈에 구분** — 계측 항목·위험 요인·연계 센서를 카드마다 시각적으로 다르게
2. **전문 매뉴얼 톤** — 홈페이지·지중경사계 페이지와 **디자인 언어 정렬** (teal accent·surface-2·16px radius)
3. **하위 호환** — 기존 `{ title, body }`만 있어도 **깨지지 않음**
4. **점진 적용** — 파일 ~120개 purpose 일괄 수정 **금지** · 파일럿 → 배치

### 2.2 비목표

- 카드마다 Figure 삽입 (hero·`sectionImages`와 역할 분리 유지)
- OEM 제품 카탈로그형 UI
- JS 애니메이션·무거운 차트 라이브러리

### 2.3 접근성·성능

- 색만으로 구분하지 않음 — **아이콘·라벨·테두리 패턴** 병행
- `prefers-reduced-motion` 존중
- 추가 HTTP 요청 없음 — **인라인 SVG 스프라이트** 또는 CSS 기하 아이콘

## 3. 목표 UI (3종 변형)

### 3.1 분야·공종 (`fields/*`) — **Purpose Panel**

```
┌─────────────────────────────────────────────────────────┐
│ [통합]  통합 계측 목적          (span 2 cols, accent)    │
│         굴착 단계별 벽체·배면·지보재…                    │
│         [지중경사계] [하중계] [토압계]  ← sensor chips   │
├──────────────┬──────────────┬──────────────┤
│ ↕ 변위       │ ⚖ 하중       │ 💧 수리      │
│ 벽체·배면…   │ 지보재…      │ 토압·수위…   │
│ [IPI][LC]    │ [LC]         │ [Piezo][GWL] │
└──────────────┴──────────────┴──────────────┘
```

- **첫 카드** `variant: 'lead'` — 2열 span · 좌측 teal 바 · 통합 목적
- 나머지 **측정 카테고리**별 accent token (`displacement` | `load` | `water` | `settlement` | `structure` | `integrated`)
- 본문 `<strong>센서명</strong>` 아래 **칩 행** (`sensors[]` nodeId)

### 3.2 센서 (`sensors/*`) — **Metric Trio**

```
┌─────────────┬─────────────┬─────────────┐
│  침하량      │  성토 제어   │  예측        │
│  ─────────  │  ─────────  │  ─────────  │
│  누적·속도   │  허용 속도   │  t–S 곡선   │
│  signal 라벨 │             │             │
└─────────────┴─────────────┴─────────────┘
```

- 3열 고정(모바일 1열) · 상단 **signal** 한 줄(측정량·판단 포인트)
- `body` 최소 2문장 또는 bullet 2개 권장
- 카드 하단 **관련 분야** 링크 1개 optional (`relatedField`)

### 3.3 계측 시스템 (`instruments/*`, 로거·원격) — **Component Strip**

- `layout: 'strip'` — 가로 스크롤 가능한 **넓은 카드** (아이콘 + 제목 + 1줄)
- 또는 2열 `matrix-card` 스타일 (지중경사계 페이지 `matrix-grid` 차용)

## 4. 데이터 스키마 (확장)

### 4.1 카드 객체 (하위 호환)

```js
// 최소 (기존) — 그대로 동작, 기본 variant만 적용
{ title: '벽체 안정성', body: '…<strong>지중경사계</strong>…' }

// 권장 (신규 필드 optional)
{
  title: '벽체·배면 변위',
  body: '<p>심도별 수평변위 프로파일…</p>',
  variant: 'default',           // 'lead' | 'default' | 'compact'
  accent: 'displacement',       // §4.2 토큰
  signal: '수평변위 · 변위속도',  // 센서 템플릿용 eyebrow
  sensors: ['sensors/inclinometer', 'sensors/displacement-transducer'],
  relatedField: 'fields/retaining-excavation/earth-retaining-wall'
}
```

### 4.2 accent 토큰 (CSS 변수 매핑)

| token | 용도 | 좌측 바 색 |
|-------|------|------------|
| `integrated` | 통합·개요 | `--teal` |
| `displacement` | 변위·경사·신축 | `--teal-vivid` |
| `load` | 하중·장력 | `--navy` |
| `water` | 수위·간극수압 | `--blue` (기존 팔레트) |
| `settlement` | 침하·처짐 | `--earth` (신규 토큰 또는 muted brown) |
| `structure` | 균열·응력·온도 | `--slate` |
| `vibration` | 진동·지진 | `--amber` |

### 4.3 섹션 수준 옵션 (선택)

```js
purposeLayout: 'grid' | 'strip'   // 시스템 허브만 strip
purposeLead: true                 // 첫 카드를 lead로 강제
```

`build-content-data.mjs`는 **passthrough** — 스키마 검증은 선택 스크립트 `validate-purpose-cards.mjs`.
