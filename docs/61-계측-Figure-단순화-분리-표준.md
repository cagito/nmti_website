# 계측 Figure 단순화·분리 표준 (정본)

**등급:** **P0 보조** — DP-01~80 **적용 시 복잡도·유형 혼합 방지**  
**역할:** [51 §3 DP-31~40](./51-계측-도면-검수-공통-원칙.md) · [§4 DIS-31~40](./51-계측-도면-검수-공통-원칙.md) 상세  
**범위:** **모든** 설치·개념·시스템 Figure — 특히 IMG-001~002·070~102

> **한 줄:** **단순화는 생략이 아니라 분리다.** 한 장에 다 넣지 말고, **틀리지 않는 작은 그림 여러 장**으로 나눈다.  
> **⛔** 원칙이 많아질수록 그림이 복잡해지면 **실패** — DP-31~40은 **더 넣지 말고 빼고 나누라**는 규칙이다.  
> **재작성 전 문서 정본** — 지금 Figure 생성·승인하지 않음.

---

## 0. 강제 지시문 (프롬프트 · 재작성 시)

```text
틀리지 않게 단순하게 그려라.

한 그림에는 하나의 계측 목적만 둔다.
주계측기는 3~5개만 넣는다.
확대도는 1개만 둔다.
설치도에는 서버·모바일·알람·데이터 흐름을 넣지 않는다.
예시 수치는 일반 기준처럼 쓰지 않는다.
데이터로거는 작게, 센서 위치는 크게 그린다.
제목을 좁게 잡고, 나머지는 별도 그림으로 분리한다.
복잡해서 한눈에 안 들어오면 그 그림은 실패다.
```

---

## 1. DP-31 — 주계측 3~5개 (SIM-01)

**금지:** 한 장에 **6개 이상** 주계측 · 전 항목 나열

**규칙:** **주계측 3~5개** · 보조는 작은 범례 또는 **별도 Figure**

**예 (096):** ① 지중경사계 ② 지표침하 측점 ③ 지하수위계 ④ 간극수압계 — 버팀보·앵커·토압은 **001/002**로 분리

---

## 2. DP-32 — 확대도 1개 (SIM-02)

**규칙:** 메인 단면 **1** + 가장 헷갈리는 부분 **확대 1**

**예 (004):** 메인 3분할 + **앵커 두부** 상세만

**금지:** 확대 2개 이상·서로 다른 계측기 설명 → **Figure 분리**

---

## 3. DP-33 — 우측 = 범례만 (SIM-03)

**금지:** 설치도 우측에 **그래프·데이터 흐름·관리기준·알림 UI** 동시 배치

| Figure 유형 | 우측 허용 |
|-------------|-----------|
| **설치 단면** | **범례** + ①~⑤ 번호 목록만 |
| **시스템도** | 데이터 흐름 (별도 Figure) |
| **그래프** | 해석 예시 (별도 Figure) |

---

## 4. DP-34 — 지층 ≤3 (SIM-04)

**금지:** 지질도가 주인공 · 지층명 과다

**권장 3층 이하:**

```text
매립층
점성토/사질토
풍화암 또는 안정층
```

정밀 지층명 → 본문·별도 지반단면도

---

## 5. DP-35 — 화살표 = 측정만 (SIM-05)

**설치도 허용:** 변위 방향 · 토압 방향 · 하중 축 · 수위(G.W.L)

**금지:** 데이터 흐름 화살표 · 4종 이상 혼합(변위·하중·수위·데이터·영향권)

---

## 6. DP-36 — 시각 우선순위 (SIM-06)

```text
1. 지표면 → 2. 지층 → 3. 구조물 → 4. 센서 위치 → 5. 측정 방향 → 6. 라벨
```

**라벨:** 2~8자 — `간극수압계` · `지표침하 측점` · 장문은 **본문**

---

## 7. DP-37 — 예시 수치 최소 (SIM-07)

**금지:** `4m` · `2H` · `±25mm` · `주의 0.3mm` 를 **일반 기준**처럼

**권장:** `근입: 안정층까지` · `H=굴착깊이` · `관리기준: 현장별` · 수치 필수 시 **`예시`·`개념`·`현장 조건별`**

(보강: [51 DIM-01](./51-계측-도면-검수-공통-원칙.md) · R-09)

---

## 8. DP-38 — 센서 80% · 계측함 10% (SIM-08)

**설치도:** 센서 위치 **크게** · **`현장 계측함`** 작게 (~10%) · 범례 ~10%

**금지:** 로거가 센서보다 큼 · 서버·LTE·모바일·알람 on 설치도 (→ **058·072** 별도)

---

## 9. DP-39 — 좁은 제목 (SIM-09)

| 나쁨 (넓음) | 좋음 (좁음) |
|-------------|-------------|
| 가시설 계측 전체 개념도 | **가시설 주변지반 계측 설치 단면도** |
| 교량 변위 계측도 | **교량 신축이음계 설치 개념도** |
| 터널 계측 종합도 | **터널 천단침하 측점 배치도** |

(연계: DP-01 · MIX-01)

---

## 10. DP-40 — 한 장에 다 설명 금지 (SIM-10)

**분리 세트 (예):**

```text
전체 개념도 1장 · 설치 단면도 1장 · 상세도 1장 · 데이터 흐름도 1장 · 그래프 예시 1장
```

**금지:** 설치+원리+데이터+경보 **한 장** — 전부 틀어짐

(연계: DP-10 · DIS-10)

---

## 11. 폐기 체크리스트 (DIS-31~40)

| # | 폐기 조건 |
|---|-----------|
| **DIS-31** | 주계측 **6개 이상** |
| **DIS-32** | 우측 패널이 메인보다 **복잡** (그래프·흐름·경보) |
| **DIS-33** | 확대도 **2개 이상**·서로 다른 계측기 |
| **DIS-34** | 설치도에 **서버·모바일·알람·데이터 흐름** |
| **DIS-35** | 지층 과다로 **센서 위치 묻힘** |
| **DIS-36** | 화살표 **4종 이상** 혼합 |
| **DIS-37** | **예시 수치**가 일반 기준처럼 보임 |
| **DIS-38** | **데이터로거 > 센서** 시각 비중 |
| **DIS-39** | **제목 과넓음**·공종 혼합 |
| **DIS-40** | **한 장**에 설치·원리·데이터·경보 전부 |

## 12. 역할별 완화 (분리 우선)

| Figure 역할 | DP-31 적용 |
|-------------|------------|
| **주변지반·단일 목적** (096 등) | **주계측 3~5** 엄격 |
| **가시설 전체 개념도** (001) | 번호·범례에 항목 나열 가능 — **시각 주인공은 3~5** · 나머지는 작은 아이콘·범례 |
| **대표 설치 단면도** (002) | ①~⑪ **번호 목록** 허용 — **우측=범례·번호만**(DP-33) · 데이터 흐름·확대 2개는 금지 |

> **원칙:** 항목 수 예외가 있어도 **한눈에 안 들어오면 R-34** — 그때는 **002 본체 + 004·030 등 분리**
