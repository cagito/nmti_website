# IMG-008 터널 전단면 내공변위 — 출판 부적합 재검수 및 구현계획

**대상 이미지:** `assets/images/technology/IMG-008.webp` / `assets/images/technology/IMG-008_터널-전단면-내공변위-측정시스템_상부아치내공변위.png`  
**대상 노드:** `fields/tunnel/convergence` hero  
**판정:** `REGENERATE` 또는 최소 `MAJOR_FIX`로 강등 후 재검수  
**우선순위:** P1 — 기술자료 hero 이미지이며 문서 간 상태 불일치가 존재함  
**작성 목적:** 현재 registry의 `PASS` 상태와 기존 오류분석 문서의 `REGENERATE` 판정 불일치를 해소하고, 출판용 Figure로 재제작·재검수하기 위한 실행 계획 수립

---

## 1. 선정 사유

IMG-008은 현재 `scripts/image-review-registry.json`상 `reviewGrade: PASS`, `visualReview.grade: PASS`, `productionMethod: ai-reviewed`로 등록되어 있다. 그러나 기존 `docs/20-IMG-008-터널-내공변위-오류분석-및-재작업-계획.md`에서는 동일 이미지에 대해 `REGENERATE` 판정을 내렸고, `audit:img008 PASS ≠ Figure 합격`이라고 명시한다.

따라서 IMG-008은 단순 저해상도 문제가 아니라 **검수 상태 불일치 + Figure 기하 오류 + 출판 게이트 불확실성**이 동시에 존재하는 이미지로 본다.

|---|---|---:|
| C1 | P6/P7 또는 Extension Tube가 건축한계 Envelope 내부·경계에 배치될 가능성 | REGENERATE |

| C3 | 노반·미계측 영역 아래에 Extension Tube 문구가 남아 하부 계측으로 오인 가능 | REGENERATE |
| M1 | 제목의 `전단면`과 실제 상부 아치 개방 체인 표현 간 혼동 | MAJOR_FIX |
| M2 | 프롬프트·PNG·본문 간 P1~P5 / P1~P7 측점 불일치 | MAJOR_FIX |
| M3 | `audit:img008`가 PNG 기하를 검사하지 않아 Figure 오류가 PASS 가능 | MAJOR_FIX |

## 3. 적용 원칙

### 3.1 제작 방식

- IMG-008은 FT-B 복합 개념도이다.
- FT-B는 `cad` 또는 `ai-reviewed` 방식만 허용한다.
- Pillow 신규 재작성, agent-svg, 1-shot AI는 금지한다.
- 최종 반영 파일은 PNG/WebP로 운영한다.

### 3.2 Figure 내용 원칙

- 계측 목적은 `터널 상부 아치 내공변위` 하나로 제한한다.
- 전둘레 폐합형, 360도 invert 계측, 천단침하계, ACE-TCS, 진동현식/VW 라벨은 금지한다.
- 건축한계 Envelope 내부는 `통행·궤도 / 미계측` 영역으로 표현한다.
- 센서 Kit, Extension Tube, Ball Joint는 라이닝 측 Envelope 외측에만 배치한다.
- 하부 노반·invert·건축한계 내부에는 센서·튜브·Kit가 없어야 한다.

## 6. 구현 단계

### Phase 0 — 상태 정리

1. registry에서 IMG-008을 재검수 대상으로 강등한다.
2. `requiresReaudit: true`를 설정한다.
3. `notes`에 `docs/20` 및 본 문서 링크를 추가한다.
4. 기존 운영 이미지는 임시 유지하되, 신규 PASS 전까지 출판 PASS로 간주하지 않는다.

### Phase 1 — 제작 프롬프트 / redline 작성

1. ImageWorks 프롬프트에 다음 고정 문구를 삽입한다.

```text
이 Figure는 터널 상부 아치 내공변위 계측 개념도이다.
건축한계 내부는 통행·궤도 미계측 영역이다.
센서 Kit, Extension Tube, Ball Joint는 라이닝 측 Envelope 외측에만 배치한다.
P1~P5 상부 아치 개방 체인으로 표현한다.
P6/P7, invert, 노반, 전둘레 폐합, 천단침하계, 진동현식/VW 라벨은 금지한다.
```

2. redline에는 측점 위치, Envelope, 노반 미계측 구역, 우측 ΔX/ΔY 패널을 분리 표기한다.

### Phase 2 — 이미지 재제작

1. 외부 AI 또는 CAD/Illustrator에서 PNG 생성.
2. 최소 1920×1080, 가능하면 2× 해상도 유지.
3. 파일명은 기존 운영 체계를 유지한다.

```text
assets/images/technology/IMG-008_터널-전단면-내공변위-측정시스템_상부아치내공변위.png
assets/images/technology/IMG-008.webp
```

4. `source/` 또는 ImageWorks 쪽에 동일명 원본·프롬프트·redline 보관.

### Phase 3 — 변환 및 빌드

```bash
python scripts/convert-technology-webp.py
npm run build:images
npm run build:content
npm run build:seo
npm run sitemap
```

### Phase 4 — 검수

다음 조건을 모두 통과해야 한다.

| 검수 | 합격 기준 |
|---|---|
| Q1 | 모든 측점·튜브·Kit가 Envelope 내부를 침범하지 않음 |
| Q2 | Envelope 캡션이 `통행·궤도(미계측)`로 읽힘 |
| Q3 | 노반·invert에 Kit/Extension Tube 문구 없음 |
| Q4 | P1~P5 상부 아치 개방 체인임 |
| Q5 | 전둘레·폐합·천단침하로 오인되지 않음 |
| Q6 | 100%·200% 확대 시 라벨·선명도 OK |
| Q7 | `진동현식`, `VW`, 특정 제품명 없음 |

### Phase 5 — PASS 승격

1. `scripts/image-review-registry.json`에서 다음 조건으로 PASS 승격한다.

```json
{
  "reviewGrade": "PASS",
  "requiresReaudit": false,
  "productionMethod": "ai-reviewed",
  "visualReview": {
    "grade": "PASS",
    "checklist": "IMAGE_AUDIT_CHECKLIST §5.1 + IMG-008 Envelope checks"
  }
}
```

2. `npm run verify:content` 또는 `npm run verify:local` 실행.
3. 운영 배포 후 `npm run verify:production` 실행.

---

## 7. 완료 기준

- 문서 간 상태 불일치가 사라진다.
- registry, IMAGE_REVIEW_LOG, images.js, content-data, 실제 PNG/WebP가 동일 상태를 가리킨다.
- IMG-008은 `PASS = 기술 게이트 + 출판 게이트 + 제작방식 정합` 조건을 충족한다.
- 향후 `audit:img008`는 텍스트뿐 아니라 Envelope 침범 여부를 검수할 수 있어야 한다.

---

## 8. 보류/주의

- 본 계획은 구현계획 문서이며, 실제 이미지 파일 교체는 외부 제작 PNG가 확보된 뒤 수행한다.
- agent-svg 또는 Cursor가 직접 SVG를 그리는 방식은 사용하지 않는다.
- Pillow 재렌더는 FT-B 출판 이미지 최종본으로 사용하지 않는다.
