### 2.4 계측센서별 — TREE 재배치 (Phase 2)

**Before**

```text
group-sensor
├ instruments/sensors (센서)
│  └ sensors/* ×17
├ instruments/datalogger
├ instruments/communication
└ instruments/power
```

**After**

```text
group-sensor
└ sensors/* (17종, 직접 나열)
   ├ inclinometer … weather-station
   └ borehole-extensometer (다점지중변위계 — 유지 권고)

group-system
├ instruments/modes        → label: 계측 방식
├ instruments/datalogger   → label: 데이터 로거
├ instruments/communication → label: 통신·전송
│  └ (원격 모니터링은 §2.5)
├ instruments/power        → label: 전원 구성
├ sensors/remote-monitoring-system → label: 원격 모니터링 (이동)
└ instruments/data-management (신설 또는 modes/smart 승격) → label: 데이터 관리
```

**코드 작업:**

1. `group-sensor.children` = `sensors/*` 노드 배열 (중간 `instruments/sensors` 제거).
2. `instruments/datalogger`, `communication`, `power`를 `group-system.children`으로 **이동** (id 불변).
3. `INSTRUMENT_SUBGROUPS` 갱신 — `instruments/sensors` 제거, `group-system` 직속 카테고리 id 추가.
4. `getAllContentNodeIds()` — 센서 그룹 제거 후에도 113노드 카운트 **동일** 확인.

**숨길 노드 (메뉴 비노출, URL 유지):**

- `instruments/modes/smart`, `instruments/modes/ai` — `tree.js` 또는 TREE `hidden: true` 플래그 도입 검토.
- 하위 설명은 `데이터 관리`·`원격 모니터링` 본문 내 링크.

### 2.5 원격 모니터링 · 데이터 관리

| 목표 메뉴 | 현재 nodeId | 작업 |
|----------|------------|------|
| 원격 모니터링 | `sensors/remote-monitoring-system` | `group-system` 직속 2단으로 이동 (parentId만 변경) |
| 데이터 관리 | 없음 (smart/ai 분산) | **Option A:** `instruments/data-management` 신설 + smart/ai 요약 · **Option B:** `instruments/modes/smart` label을 `데이터 관리`로 변경하고 ai는 smart 하위 링크 |

**권장:** Option A — smart/ai SEO 페이지 유지, 허브만 신설.

### 2.6 댐·제방 하위 (Phase 2b — 콘텐츠 밀도 판단)

| 현재 9 리프 | 목표 6 그룹 | 병합안 |
|------------|------------|--------|
| leakage | 침투·누수 | `leakage` 유지 |
| pore-pressure | 수위·수압 | `pore-pressure` + 지하수위 교차 |
| settlement + displacement | 침하·변위 | 리프 2개 유지 또는 카테고리 페이지 앵커 |
| strain + tilt | 응력·변형률 | `strain`·`tilt` 유지 |
| temperature + seismic | 온도·지진 | 리프 2개 유지 |
| river-levee | 하천제방 | 유지 |

**판단 기준:** 각 리프 `content-data` 분량·hero PASS 여부. 9페이지 모두 PASS이면 **1차는 label만 정리**, 메뉴 축소는 **2차** (단일 `fields/dam` + 섹션 앵커).

댐 본문 권장 섹션 순서는 프롬프트대로 `categories.mjs` `fields/dam` sections 순서 재정렬.

## 6. 최종 확인 체크리스트 (프롬프트 대응)

```text
✅ 상위 메뉴 3개: 구조물·공종별 / 계측센서별 / 계측 시스템

✅ 센서명이 계측센서별 2단에 직접 노출
✅ 데이터 로거·통신·전원·원격 모니터링이 계측 시스템 소속
```
