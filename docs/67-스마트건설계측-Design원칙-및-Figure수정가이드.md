## A. 웹 Figure·도면 수정 가이드 (Cursor용)

### A-1. 배면 계측기 이격·벽체 매설 (SOE-INST-01 · P1)

| 항목 | 반영 내용 | 등급 |
|------|-----------|------|
| **위치** | **센서형 다단식 지중경사계**·**지하수위계**·**간극수압계** = **배면 지반** 천공 — 흙막이·CIP **벽체 외측** 영향권 | P1 |
| **금지** | **CIP·콘크리트 벽체 단면 내부**에 관로·센서 **직접 매설** | P1 → REGENERATE |
| **이격** | 벽체에 **밀착**·관로가 벽체를 **관통**하는 표현 금지 — **`벽체 외측 배면`** · **`설계 이격거리`**(현장 조건별) | P1 |
| **수치** | `0.5m 이상` 등 **고정 m**을 모든 Figure에 쓰지 **않음** — redline·특정 현장만 설계값 인용 | DIM-01 |

**Cursor 한 줄:** `지중경사계·지하수위·간극수압은 CIP/벽체 내부가 아니라 배면 지반 천공. 벽체 외측·설계 이격.`

→ [INSTRUMENTATION §3.1.2](./INSTRUMENTATION_DRAWING_RULES.md)

### A-2. 설치 심도·안정층 근입 (IPI-EMBED · 기존 유지)

| NotebookLM | 저장소 반영 |
|------------|-------------|
| 부동층(기반암) **1.5m 이상** 근입 | **Base = 영향 심도 하부 안정층**까지 — **깊이는 설계·지반조사·계획서** · Figure에 **임의 m 일반화 금지** |
| 관이 중간에 끊김 | **P1:** 케이싱·관로 **GL 연속** · Base 아래 **안정층 맥락선** |

### A-3. 지표침하계 주변 표현 (SETTLE-PLATE-01 · P1)

| 금지 | 올바른 표현 |
|------|-------------|
| 침하판 주변 **콘크리트 타설·콘크리트 해칭** | **다져진 토사** · 지표면과 동일 연속 |
| 침하판을 **구속**하는 주변 구조 | 침하판·연장봉 **연직 자유 거동** · ↓ 변위 화살표 |

**Cursor 한 줄:** `지표침하계 주변은 다져진 토사. 콘크리트 패드·해칭으로 침하판 구속 금지.`

### A-4. 터널 막장·자동 측점 (TUN-FACE-01 · P1)

| NotebookLM | 저장소 반영 |
|------------|-------------|
| 막장 **10m 이내** 타겟 | **막장 영향권·초기변위 구간**에 측점/프리즘 — **심도·거리는 계획서** (고정 10m Figure 기준 **아님**) |
| 조감도에서 막장과 동떨어진 타겟 | 막장·전방지반 **선행변위** 맥락과 **인과**가 보이게 — `fields/tunnel/face-advance` |

**금지:** 막장과 무관한 원거리 타겟만 나열 · 내공변위(008)와 혼합

### A-5. 스마트 계측 **시스템 구성도** (FT-C만 · DP-61)

| NotebookLM | 저장소 반영 |
|------------|-------------|
| 태양광 패널 아이콘 | **시스템·전원 블록도**(IMG-047·066·058 등) — **설치 단면도에 혼입 금지** |
| 통신 **무선 점선** · LoRa/LTE | **communication** 토폴로지 — `LTE M2M`·`LoRa` 용어 ([13](./13-LTE-M2M-용어-및-Figure-개편-계획.md)) · **`1km 이상` 고정 표기 금지** |
| 데이터 흐름 | `센서 → 로거 → 모뎀/안테나 → 서버 → 대시보드` — **설치도 ≠ 시스템도** ([65](./65-계측-Figure-유형-분리-레이아웃-표준.md)) |

**Cursor 한 줄:** `태양광·무선·서버는 시스템 블록 Figure만. 흙막이 단면도에 넣지 마라.`

## B. 스마트 건설계측 Design 원칙 (SD-01~08)

> **범위:** 장비·펌웨어·운영 **아키텍처** · 본문·modes 노드 · **FT-C 시스템 Figure**.  
> **Figure P0와 분리** — ±0.1°·상보필터 등은 **제품 성적서·시방** 인용 시에만.

| ID | 원칙 | Figure·콘텐츠 반영 | NotebookLM 대응 |
|----|------|-------------------|-----------------|
| **SD-01** | **데이터 신뢰성** | 노이즈·온도·설치 상태를 **품질검증 단계**로 표현 ([59 DP-12](./59-계측-운영-모드-구조-환경-AI-표현-표준.md)) | ±0.1°·상보필터 → **「설계 정밀도·보정 알고리즘」** 문구, 수치 일반화 X |
| **SD-02** | **현장 내구·방수** | 지하수위·매몰 환경 = **실링·보호관·함체** 표현 (설치도) | 패키징·실링·부식 방지 — 일반 원칙 |
| **SD-03** | **설치·유지보수성** | **브라켓·보호함·탈착 가능 마운트** — 제품명(자석 등) **Figure 라벨 금지** | 네오디움 자석 → **「전용 브라켓·현장 조건별」** |
| **SD-04** | **실시간·원격 수집** | **1:N 무선** = 시스템도 · 수동 7~10일 공백은 **운영 방식 비교** 문구만 | LoRa/LTE — 고정 거리 X |
| **SD-05** | **경보·운영 UI** | 대시보드 = **단위·시간·기준·상태·조치이력** ([59 DP-19](./59-계측-운영-모드-구조-환경-AI-표현-표준.md)) — AI 자동 위험 확정 **금지** | 모바일 경보 — 스마트 **운영** 블록 |
| **SD-06** | **여유·중복(Redundancy)** | 계획서·본문 — **주요 측점 이중화** · 매립 vs 노출 **병용**은 현장별 | 여유분 센서 — 콘텐츠·계획서 |
| **SD-07** | **디지털 연계** | **3D·디지털 트윈** = **시스템·운영 모드** Figure(094~102) — **단면 설치도 혼입 금지** | 디지털 트윈 — 아키텍처 지향 |
| **SD-08** | **KCS·설계도서 우선** | 기술 주장·수치 = **registry·설계도서** — NotebookLM 수치 **그대로 복사 금지** | 국가건설기준 준수 |

## D. Cursor 프롬프트 보조 (Figure 유형별)

**흙막이·주변지반 단면 (IMG-002·096):**

```text
지중경사계·지하수위·간극수압은 배면 지반 천공. CIP/벽체 콘크리트 내부 매설 금지. 벽체 외측·설계 이격.
IPI Base는 안정층까지 — 임의 m 수치 금지.
```

**지표침하계 상세 (IMG-032 등):**

```text
침하판 주변 = 다져진 토사. 콘크리트 해칭·패드로 침하 구속 금지.
```

**시스템·전원·통신 (IMG-047·058·066):**

```text
소형 태양광·배터리·LTE/LoRa 모뎀·안테나. 무선 구간은 점선. 고정 통신거리 km 표기 금지.
설치 단면이 아니면 서버·대시보드 블록 허용.
```
