실시간·이벤트 계측

실시간·이벤트 계측의 구성, 설치, 운영, 데이터 흐름 및 현장 유지관리를 정리한 기술 자료입니다.

1. 시스템 개요

  • 목적실시간·이벤트 계측 모드는 발파, 급격한 변위, 굴착 단계 변화 등 **Trigger Event** 발생 시 초당 수십~수백 Hz로 고속 샘플링하고 즉시 전송하는 동적 모드입니다. 동적 데이터로거·진동계·트리거 DAQ가 대표 장비입니다.
  • 적용 범위타임라인 한 지점에서 **조밀한 샘플링선**·**impulse/spike 파형**·경보 대시보드로의 즉시 전송이 시각 핵심입니다. 번개·폭발 CG는 사용하지 않습니다.

2. 구성 요소

  • 고속 샘플링Trigger 후 연속 고주파 수집
  • 이벤트 포착발파·급변·임계 초과 순간 기록
  • 즉시 전송동적 벡터·파형 실시간 업로드
  • 대조normal-mode 평탄 추세와 명확히 구분

3. 현장 구성·원리

  • 구성 흐름이벤트 발생: 트리거(진동·임계·일정) → 동적 DAQ/진동계(고속 샘플링) → 버퍼 저장 → 실시간 서버·경보. 종료 후 normal-mode 복귀.
  • 측정 대상트리거는 하드웨어(진동 threshold)·소프트웨어(관리자 명령)·일정(발파 시각)로 설정합니다.
  • 측정 원리Pre-trigger buffer·Post-trigger record로 이벤트 전후 파형을 보존하고 PPV·변위 peak를 산출합니다.

4. 설치·운영

  • 이벤트 유형 · 샘플링 rate · duration을 정의
  • 동적 DAQ · 트리거 채널 · GPS 시간 동기 설정
  • 발파 · 굴착 일지와 트리거 시간 태그 연동
  • 실시간 대시보드 · 경보 연동을 테스트
  • 이벤트 종료 후 normal-mode 복귀 규칙 설정

자주 묻는 질문

정적 로거로 충분한가요?

발파·충격 이벤트는 동적 DAQ 또는 이벤트형 진동계가 필요합니다. 정적 Scan Interval만으로는 파형을 놓칩니다.

터널 발파(097)와?

097은 현장 PPV·영향권 Figure입니다. 본 모드는 **시스템 토폴로지**·고속 샘플링 전환(095)입니다.

normal-mode와 차이?

normal-mode는 안정 구간 고정 주기, realtime-mode는 이벤트·위험 구간 고속·트리거 수집입니다.

전환 기준은?

관리기준 비율·발파 일정·담당자 수동 전환 등을 계획서에 정의합니다.
