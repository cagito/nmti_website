자동 계측

자동 계측의 구성, 설치, 운영, 데이터 흐름 및 현장 유지관리를 정리한 기술 자료입니다.

1. 시스템 개요

  • 목적자동계측(KCS 데이터 수집방법)은 센서 데이터를 설정 주기로 자동 수집·저장하고, 필요 시 유선·무선 통신을 통해 서버로 전송하여 표출·경보·보고 체계와 연계하는 운용방식입니다.
  • 적용 범위KCS에서는 구간 특성에 따라 수동계측에서 자동계측으로 전환할 수 있으며, 실시간 감시·자동경보, 원격지·접근 곤란, 센서 다수·인력 부족 등에서 자동화를 검토합니다. 현장 데이터로거는 자동 수집의 핵심 장비이나, 자동계측 전체를 로컬 저장만으로 한정하지 않습니다.

2. 구성 요소

  • 연속 수집야간·휴일·고빈도 데이터
  • 객관성측정자 편차 감소
  • 연계통신·표출·경보·보고

3. 현장 구성·원리

  • 구성 흐름센서 → 데이터로거 → 로컬 저장 → (선택) 유선·무선 통신 → 서버·표출·경보. 무인 현장은 태양광·배터리·방수 함체와 함께 스캔 주기·경보 규칙을 설정합니다.
  • 측정 대상센서 → 데이터로거 → 로컬 저장 → (필요 시) 통신 → 서버·표출·경보·보고 순으로 운용합니다. 샘플링 주기·필터·캘리브레이션이 데이터 품질을 결정합니다. 무인 현장 전원은 태양광·배터리 설계가 핵심이며, AC 가능 구간은 전원 체계 전체를 함께 검토합니다.

4. 설치·운영

  • 로거 채널 · 센서 매핑
  • 전원 · 태양광 · 배터리 설계
  • 수집 주기 · 경보 조건 설정
  • 초기치 · 영점 안정화
  • 수동계측 백업 절차 유지

자주 묻는 질문

자동과 원격의 차이?

자동계측은 KCS 수집방법(현장 수집·저장·연계)입니다. 원격 자동계측은 통신·서버·대시보드까지 포함한 NMTI 운영 확장 개념입니다.

로거만 있으면 자동계측?

로거는 핵심 장비입니다. 자동계측 운용은 수집·저장과 필요 시 전송·표출·경보·보고까지 포함합니다.

스캔 주기는?

위험도·KCS·관리기준에 따라 분~시간 단위를 설정합니다. MUX 사용 시 채널당 유효 주기를 검증합니다.

수동 병행?

이상 작동·통신 장애 대비 수동 측정을 계획서에 포함합니다.
