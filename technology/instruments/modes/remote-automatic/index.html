원격 자동계측

원격 자동계측의 구성, 설치, 운영, 데이터 흐름 및 현장 유지관리를 정리한 기술 자료입니다.

1. 시스템 개요

  • 목적원격 자동계측은 KCS 계측방식 분류가 아니라, 현장 자동계측 데이터를 LTE M2M·유선 등으로 서버에 전송하고 웹·모바일에서 모니터링·경보하는 NMTI 운영 확장 계층입니다. 원격계측시스템이 대표 구현 형태입니다.

2. 구성 요소

  • 실시간성현장 없이 추세·경보 확인
  • 다현장중앙 통합 모니터링
  • 보고자동 그래프·보고서

3. 현장 구성·원리

  • 구성 흐름현장 자동계측 → LTE M2M·게이트웨이 → 서버·DB → 웹·모바일·경보. 가용성·보안·데이터 품질이 핵심이며 수동 백업 계획을 병행합니다.
  • 측정 대상센서 → 로거 → LTE M2M 모뎀(또는 게이트웨이) → 서버·DB → 웹/모바일 → 경보의 계층 구조입니다. 가용성·보안·데이터 품질 관리가 핵심입니다.

4. 설치·운영

  • 아키텍처 · 보안 설계
  • 현장 로거 · 통신 설치
  • 서버 · DB · API 구축
  • 대시보드 · 권한 · 경보 규칙
  • 운영 · DR · 백업 절차

자주 묻는 질문

원격계측시스템과 동일?

원격 자동계측은 방식(개념)이고, 원격계측시스템은 이를 구현하는 통합 장비·플랫폼입니다.

수동 병행?

통신·센서 장애 대비 수동 측정 계획을 유지하는 것이 KDS·KCS 원칙입니다.

대시보드가 필수?

원격 자동계측은 서버·웹·모바일 등 모니터링 수단이 포함됩니다. 규모에 따라 단순 알림부터 확장합니다.

LTE만으로 충분?

단일 현장·소규모는 가능하나, 다현장·게이트웨이·보안 요건에 따라 구성을 확장합니다.
