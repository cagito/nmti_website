경보·알림 상태

경보·알림 상태의 구성, 설치, 운영, 데이터 흐름 및 현장 유지관리를 정리한 기술 자료입니다.

1. 시스템 개요

  • 목적경보·알림 상태는 센서 값이 **관리기준치(주의·경고·위험/조치 3단계)** 를 초과했을 때 시스템이 비상 상태를 감지하고 **현장 경광등**·**SMS/푸시**·서버 알림을 즉시 발송하는 메커니즘입니다. SF형 경고창·네온 일러스트는 사용하지 않습니다.
  • 적용 범위그래프 상 **수평 threshold line**·데이터 plot의 **돌파 변곡점**·경광등·알림 블록으로 연결되는 **제어 흐름도**가 hero(102)의 핵심입니다. 스마트 계측·IMG-

2. 구성 요소

  • 3단계 기준Caution·Warning·Action 수평선
  • 즉시 전파경광등·SMS·푸시 트리거
  • 조치 연계공정 중지·현장 확인·보고
  • 로그경보 이력·확인·해제 기록

3. 현장 구성·원리

  • 구성 흐름계측값 → 기준 비교(로거/서버) → 임계 돌파 시 경광등·SMS/푸시·Escalation. 해제는 확인 측정·책임자 승인 후 처리합니다.
  • 측정 대상로거 또는 서버 규칙 엔진이 실시간·배치 데이터를 기준과 비교합니다.
  • 측정 원리Action 초과 시 Escalation chain(현장→감리→발주)으로 알림이 전달되며, 해제는 확인 측정·책임자 승인 후 처리합니다.

4. 설치·운영

  • 측점별 Caution/Warning/Action 임계값 설정
  • 현장 경광등 · 무선 알림 수신기 배치
  • SMS · 이메일 · 앱 푸시 수신자 목록을 관리
  • 경보 · 확인 · 해제 워크플로를 문서화
  • 오경보 · 통신 장애 시 fallback을 정의

자주 묻는 질문

IMG-054와 차이?

054는 정상→주의→경고→위험→조치 **프로세스** Figure입니다. 102는 threshold 돌파→경광·SMS **제어 흐름** hero입니다.

스마트 계측(073)?

073은 플랫폼·경보·보고 통합 개념입니다. alarm-status는 **기준 초과 비상 상태** 전용입니다.

유지관리 75%·90% 단계는?

설계예상·최대허용 대비 단계별 관찰·경보입니다. 대구 3호선 통합 유지관리 S/W·SMS와 같은 운영 맥락입니다.

normal-mode와 차이?

normal-mode는 안정 구간 정기 수집, alarm-status는 기준 초과·비상 경보·SMS·경광 등 즉시 조치 상태입니다.
