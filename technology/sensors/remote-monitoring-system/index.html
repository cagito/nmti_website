원격 모니터링

원격 모니터링의 측정 목적, 적용 현장, 설치, 원리, 데이터 해석 및 관련 공종을 정리한 기술 자료입니다.

1. 시스템 개요

  • 목적원격계측시스템은 센서, 데이터로거, 통신, 서버, DB, 웹 대시보드, 모바일 경보를 통합해 현장 계측을 실시간·주기 관리합니다. 자동화계측의 완성 형태입니다.
  • 적용 범위구성: 센서 → 데이터로거 → 게이트웨이/LTE → 서버 → DB → 웹 대시보드 → 모바일 알림 → 보고서
  • 주요 구성적용 현장: 댐·제방·터널·철도 건설중 계측 현장에서 다측점·다공정 데이터를 24시간 수집·경보합니다. 축조·굴착·道床 시공 단계의 QC·일·주 보고와 연동합니다.

2. 구성 요소

  • 통합다현장·다센서 일원화
  • 경보SMS·이메일·단계별
  • 보고자동 보고서·그래프
  • 품질이상값·센서 상태 감시

3. 현장 구성·원리

  • 구성 흐름센서 → 데이터로거 → 게이트웨이/LTE M2M → 서버·DB → 웹 대시보드 → 모바일 경보·보고서. 다현장은 중앙 플랫폼에서 측점·기준치·권한·이력을 통합 관리합니다.
  • 측정 대상단순 수집이 아닌 기준치·검증·이력·보고 자동화까지 포함.
  • 측정 원리IoT 게이트웨이·클라우드·온프레미스 혼용.

자주 묻는 질문

자동화계측과 동일?

원격계측시스템은 통신·플랫폼·경보까지 포함한 상위 개념입니다.

보안?

VPN·암호화·접근통제. OT·IT 분리 검토.

1차·2차 관리기준 차이는?

1차는 설계예상·초기 관찰 단계(추세·보고 강화), 2차는 최대허용·계약 한계 접근 시 경보·조치 검토 단계입니다. 75%/90%는 유지관리 사례의 비율 예시이며 절대값은 설계·계측관리계획서를 따릅니다.

로거만 있으면 원격?

현장 수집에 통신·서버·대시보드·경보가 더해져야 원격계측시스템입니다.

플랫폼과 차이?

원격계측시스템은 하드웨어·통신·서버·웹까지 포함합니다. 스마트 계측은 경보·보고 운영 계층입니다.
